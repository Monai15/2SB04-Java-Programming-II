Feature ที่เพิ่ม 2 อย่าง

1.Invalid Exeption

ระบบการจัดการข้อผิดพลาด (Error Handling) แบบ OOP ที่ดีขึ้น โดยใช้ Custom Exception ชื่อ InvalidQuestException เพื่อจัดการกรณีที่ผู้ใช้ป้อน Quest ID ที่ไม่ถูกต้องหรือไม่พบในระบบ แทนการ return null และเช็ค if ในโค้ดเดิม

วิธีทดสอบฟีเจอร์ใหม่

- Case ที่อินพุตถูกต้อง - เลือก 1 เพื่อแสดงรายการของไอดีที่มี เลือก 2 และพิมพ์ไอดีตามที่แสดง

  ผลลัพธ์ที่คาดหวัง: โปรแกรมทำงานในส่วนถัดไปซึ่งคือการแสดงคะแนนที่ได้รับ

- Case ที่อินพุตผิดพลาด - เลือก 1 เพื่อแสดงรายการของไอดีที่มี เลือก 2 และจงใจพิมพ์ไอดีผิด

      ผลลัพธ์ที่คาดหวัง: มีข้อความเตือนผู้ใช้ว่าไอดีไม่ถูกต้องพร้อมแสดงสิ่งที่ผู้ใช้พิมพ์

  2.Save Stat into CSV format
  ฟีเจอร์ที่เพิ่มเข้าไปในโปรแกรมคือความสามารถในการบันทึกข้อมูลสถิติ ลงในไฟล์ CSV หลังจากกด quit โดย method ชื่อ saveToCSV จะถูกเพิ่มใน QuestService เพื่อ export ข้อมูล completed, failed จาก HashMap stats และ totalScore เป็นไฟล์ โดยกำหนดให้ชื่อไฟล์คือ stats.csv หากมีไฟล์อยู่แล้วก็จะทำการ overwrite ข้อมูลเก่า

วิธีทดสอบฟีเจอร์ใหม่

1)ทดลองทำ task 1 ครั้ง

2)กดเลข 3 เพื่อตรวจสอบ stat

3)กดเลข 4 เพื่อจบการทำงานของโปรแกรม

4)ตรวจสอบว่ามีการสร้างไฟล์หรือไม่

5)ทดสองทำ task 2 ครั้ง

6)ตรวจสอบว่าการ overwrite ทำงานถูกต้องหรือไม่
